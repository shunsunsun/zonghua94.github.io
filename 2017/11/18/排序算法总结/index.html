排序算法总结

发表于 2017-11-18 | 分类于 C++ |
整理一波常见的排序算法

好久不碰算法，已经对基础的排序算法淡忘了，特意整理一下，方便自己复习和理解。

选择排序

选择排序的思路比较简单：从数组中依次从第k个到第n个元素中选择最小的元素，放在第k个位置上（用swap的方式进行交换，因此是一种不稳定的排序）。
由于选择排序没有任何剪枝操作，所以时间复杂度为：最好、平均、最坏：O(n^2^)

插入排序

插入排序的思想是：将第k+1个元素放在前k个序列(已排好序)中的适当位置。由于数组的插入的时间复杂度为O(n)，并且由于前面的序列已经是排序过的，所以存在剪枝条件：任何一个元素大于第k+1个元素。
所以插入排序的时间复杂度受到元素的排列程度影响：最好O(n)：平均O(n^2^)：最坏：O(n^2^)

希尔排序

我理解的希尔排序其实是插入排序的升级改良版本。在插入排序中存在这样一个问题：如果是一个升序序列排列成降序序列(插入排序的最坏情况),那么每次都要把最后一个元素一点点移动到第k个位置，这样很耗时间。如果把序列完全打散，那么插入排序的最好情况又不复存在了。所以如果把小的元素先成块的移动，排列成一个基本有序的序列之后再把序列按照插入排序的方法进行排序应该可以省出很多运算。
希尔排序就一部分解决了这一问题，首先进行间隔很大的插入排序(插入排序的间隔为1)，然后依次缩短间隔直至1，得到最终结果。
···c++
void shell_sort(vector& nums) {
//先确定间隔值，间隔值以1, 4, 13, …
int H = 1;
while (H < nums.size() / 3) {
H = H * 3 + 1;
}
while (H >= 1) {
for (int i = H; i < nums.size(); ++i) {
for (int j = i; j > 0 && nums[j] > nums[j - H]; j -= H) {
swap(nums[j], nums[j - H]);
}
}
H /= 3;
}
}
//注意到如果H如果初始为1时，就变成了插入排序

快速排序

我将快速排序的方法看做归并排序的一种有代价的优化方式不知道对不对，但是我是按照这种方式理解的。
归并排序中存在着空间复杂度为O(n)的问题，而且随着递归的增多，耗费的空间也随之增高，那有没有什么方法既可以耗费常数个空间还可以达到O(nlogn)的复杂度呢？
不如从归并排序的方法上进行优化，归并排序的空间耗费很大程度上是由于两个数组合并时由于不确定两次的合并顺序而发生的。那么如果每次合并的过程都能保证左边就是比右边小那这个问题不就解决了么！
因此我们排序的思路就变成：

  1. 找到中间数的位置
  2. 递归排序左边和右边
  3. 以左边->中间数->右边的方式合并

但是似乎中间数很难判断，因此我们将此过程进行修改：不找中间数，而是随机确定一个数的位置，也就形成了快速排序算法。
然而，随机的确定一个数的位置这种事情随机性很大，因此就存在了最好O(nlogn)、最坏O(n^2^)、平均O(nlogn)时间复杂度。

为了解决快速排序发生最坏的情况，提出了随机打散或者从左中右三个数中取中间大数作为哨兵进行排序。
