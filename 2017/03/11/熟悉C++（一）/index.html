熟悉C++（一）

发表于 2017-03-11 | 分类于 C++ |
C++的变量和基本类型相关，主要是指针、引用、限定符const，和新标准库的auto、decltype。

准备深入学习C++，所以用博客记录一周以来看到的，之前没有印象的东西，加深自己的记忆。

基础知识

  • main函数的标准返回值为int类型的0，但是在实际使用中只有返回类型不为int才会报错，返回-1对程序没有影响。
  • C++中初始化和赋值是两个概念，要记得区分开。
  • 声明与定义的区别：声明：extern int i; （不申请存储空间）
    变量只能被定义一次，但可以声明多次。
  • C++变量名不能连续出现两个下划线，也不能由下划线+大写字母开头。（变量名可以为单下划线“_”，但是不能包含C++中的关键字，如：1_or_2 是不合法的）
  • 变量只对生命的作用域有效，且小作用域会覆盖大作用域。

变量和基本类型

引用

表示把refi与i绑定到了一起，相当于refi和i都指向了同一个地址。

  • 引用只能对对象使用，引用本身不是一个对象，所以不能对引用进行引用
  • 引用定义时必须初始化，且定义好后不能改变

指针

  • 指针和引用一样，只能对对象使用，所以不存在引用的指针
  • 空指针用nullptr（int *i = nullptr）

  • 不能把int变量的值直接赋给指针

  • void类型的指针只能看做内存空间的地址，不能取出它的值（因为不知道该取出多少个地址的值，或者有没有符号）

const限定符

必须初始化

仅在当前文件中有效，但是合一使用extern实现共享

const的引用

  • 想要对常量进行引用时，必须用常量的引用形式（引用也要加上const限定符）
  • 允许const引用一个数字变量

const的指针

  • 不能用一个普通的执政指向const常量（指针必须用const限制）
  • 允许const指针指向一个变量
  • const的限定方式

顶层/底层const

顶层const：限制的类型与变量类型相同
底层const：引用、指针(所有的引用都是底层，指针不一定)

拷贝时，const的限制条件（可以理解为优先级）
无const可以拷贝底层，底层可以拷贝顶层，即：无const>底层>顶层/底层

常量表达式

在编译时就能得到结果的表达式

constexpr常量

constexpr定义的一定是常量表达式，并且允许定义一种特殊的constexpr函数，这种函数能在编译时就知道值

  • constexpr智能定义字面值类型
  • 限定符constexpr仅仅对至指针有效

typedef与using

  • const限定

  • using，C++11新特性

auto：根据表达式自动匹配类型

  • const限定：会忽略顶层const，保留底层const
