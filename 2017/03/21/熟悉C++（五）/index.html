发表于 2017-03-21 | 分类于 C++ |
C++的函数，主要包括函数的重载，函数匹配，函数指针

基础

  • 函数中创建的变量在函数调用结束之后就会释放，如果希望知道程序终止才被销毁，需要在函数中用到局部静态对象。（在函数中创建这个对象也就是说进行了多次创建一个变量类型么？？还是说发现了已经定义过就忽视了那一句话呢？？？）

  • 参数传递：分为传值调用和传引用调用
    传值调用：实际操作过程中进行了一次参数拷贝的过程。（也可以传递指针形参，但是C++中最好使用传引用调用代替这个方式）
    传引用调用：使用传引用调用可以在函数里面改变传入的数值，而且对于参数很复杂的函数，传引用调用可以避免一次参数的拷贝过程，可以节省时间和空间。
    注意：当创建的函数中函数的形参为引用类型就要遵守引用类型的对象初始化的一切要求（比如普通引用类型不能用字面值初始化），所以尽量使用常量引用（如果不需要在函数中改变形参的值）。

  • 传递含有可变形参的函数：使用initializer_list标准库类型（感觉就是一个vector类似的容器啊），把所有要传入的参数全都放到容器里面，到时候直接把容器穿进去就可以了。但是和vector不同的是initializer_list中的元素永远是常量值，无法改变其中的元素。

返回引用值

函数的返回值的过程相当于新建了一个临时对象保存函数的值，然后在主函数中对这个临时变脸进行操作。

  • 返回值类型为引用类型有什么用途呢，最主要的用途就是可以把函数的返回值作为操作符的左值，

  • 但是要注意不能返回在函数中创建的变量的引用，包括返回一个字面值的引用，因为这些值在函数完成后会释放掉，引用的值也会有错误。

函数重载

  • 重载的原则：不允许除了返回类型之外其他类型都相同，对形参来说有没有顶层const是相同的，但是可以通过底层const来重载不同的函数，这时会通过精确匹配的方式匹配对应的重载函数。

  • const_cast对重载函数的用处

  • 重载作用域：如果在一个函数的内部对这个函数外部的函数进行重载，那么在这个函数内部使用这个函数的时候函数外的所有同名函数将都会起到任何作用。原因：一旦编译器在当前作用域找到了所需的名字，编译器就会忽略掉所有外层的同名实体。

  • 重载函数的匹配：选取最完美的匹配，如果两个函数都不是完美匹配还无法分出优劣，那么函数调用失败，因为具有二义性。
    匹配优劣判断等级：

  1. 精确匹配（可以允许数组或函数类型转换成对应的指针类型，允许添加或删除顶层const）
  2. 底层const转换实现的匹配
  3. 类型提升实现的转换（short->int）
  4. 算术类型装换或指针装换
  5. 类类型转换？？

如果在定义函数的时候使用了默认实参，那么在调用函数时，可以对这些参数进行缺省，它们会默认传入默认实参：

  • 因为对一个函数多次声明是合法的，所以可以多次添加默认实参，但是注意不能修改已经存在的默认实参

  • 变量作为默认实参：局部变量是不能作为默认实参的（因为到时候找不到那个已经被释放的局部变量的地址内容），而且默认实参应该是记住了那个变量的地址中的内容。

内联函数和constexpr函数

内联函数

一般来说，使用函数的操作过程要比直接写在一个程序中慢（要存各种寄存器一类的东西），但是内联函数在编译运行时相当于直接在原函数中将内联函数展开，也就是说不需要保存寄存器一类的操作了。定义方法：定义函数之前加上inline

constexpr函数

原则：函数的返回值类型和所有形参的类型都必须是字面值类型，而且函数体中有且只能有一条return语句。
constexpr函数被隐式的定义为内联函数
