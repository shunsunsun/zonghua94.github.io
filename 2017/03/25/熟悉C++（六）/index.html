  • 但是要注意，在大括号内赋初值和使用构造函数初始值的编译过程是完全不同的。构造函数初始值是对类的内部成员赋初始化之后将类实例化；而大括号赋初值原理相当于先对类中元素默认初始化再执行构造函数执行赋值操作。

成员函数与内联函数

定义在类内部的函数是隐式的inline函数。
我们既可以在类内部声明的时候显式的声明内联，也可以在定义的时候把函数定义为内联。

类的成员

可变数据成员

通过mutable定义一个可变数据成员之后，就算这个类的对象是const的，也可以对这个成员进行修改，写入。

静态成员

不需要对类进行初始化就能使用的值，而且所有的类共有这同一个值，一个类中修改，所有的会全部修改。

  • 一般来说不能在类的内部初始化静态成员，一般像全局变量一样在类的外部声明初始化静态成员（怎么像是函数一样，既要定义，还要再进行一遍定义？），而且就算静态成员和静态函数是私有的，在类的外部也可以使用？还是说只有对静态私有成员的定义初始化能够使用类的静态函数呢？

  • 允许在类的内部对静态成员提供一个const整数类型的类内初始值，而且初始值必须是一个常量表达式。

  • 与类内普通成员的区别：静态成员可以是不完全类型，而且静态成员可以作为类内部成员的默认实参。

类的访问控制

  • public与private
  • class和struct都可以表示一个类，但是他们的区别在于他们的默认访问权限不一样，使用struct关键字，则默认的访问说明符是public，使用class关键字，默认的访问说明符是private
  • 在类内部可以定义一个类型成员（使用typedef和using），同样可以使用访问控制符对他们控制，要注意的是：用来定义类型的成员必须先定义后使用（和普通成员有所区分）

友元

如果想要其他的某些函数或者类使用这个类内部的private成员，可以在这个类中添加一条以friend关键字开始的函数声明语句即可。（相当于给了这个函数或类使用本类的内部成员的权限，不受访问控制级别的约束）

  • 友元不存在传递性
  • 使用友元的时候要注意设计程序结构的时候声明和定义的依赖关系：
  1. 首先定义 Window_mgr类，声明clear函数，但是不能定义（因为没有和Screen建立友元关系）
  2. 定义Screen，包括对友元的声明
  3. 定义clear，此时才能使用它访问Screen成员

类的类型

要记住，就算是内部结构完全相同的类，他们的类型也是不同的

聚合类

原则：所有的成员都是public；没有定义任何构造函数；没有类内初始值；没有基类和virtual函数。（相当于用一个类保存了很多变量）

  • 这样的累可以通过初始值列表的方式进行初始化，初始值列表中的赋值初始化顺序为类中成员的声明顺序

字面值常量类

原则：数据成员必须是字面值类型；至少含有一个constexpr构造函数；数据成员的初始值必须是一条常量表达式；用默认的析构函数

  • 至少有一个constexpr构造函数，意味着可以有其他的函数。
  • constexpr构造函数函数体是空的，没有返回值。

名字查找和作用域

类的编译过程

  1. 编译成员声明
    1. 在成员函数内，改名字出现之前进行查找
    2. 在类内查找，此时类内的所有成员都被考虑
    3. 在类定义之前的作用域继续查找
  2. 编译函数体

对类型名的定义

在内层作用于允许重新定义外层作用于的名字，但是类中不允许重新定义，而且就算是定义一个完全相同的也不可以。
