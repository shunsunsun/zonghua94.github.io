C++各种数据结构以及一些操作

发表于 2018-12-16 | 分类于 C++ |
vector, list, set, map, priority_queue 的基础知识

现在虽然掌握了C++中的数据结构的基本使用，但是不知道其实现的原理，不知道插入删除查找操作的基本事件复杂度，所以有点好奇，稍微研究了一下，总结了一些东西。

vector

vector数据结构比较简单，就是和数组基本类似（似乎在申请vector的过程中就是在申请数组，当数组装不下这些元素的时候，就申请一个二倍空间的数组，将原来的数组复制进去）
因此时间复杂度和数组相同：
insert: O(N)
erase: O(N)
find: O(1)

list

list就是以双向链表实现，时间复杂度也很好理解
insert: O(1)
erase: O(1)
find: O(n)

priority_queue

priority_queue
优先队列内部是以堆的方式实现的，并且Functional类可以定义排列的方式（大顶堆，小顶堆，自定义数据类型的比较），堆得复杂度为：
insert: O(N)
erase: O(N)
find: O(1)

set

set
集合内部是以红黑树实现的（平衡的二叉排序树），Compare类可以定义排列的方式，复杂度为：
insert: O(logN)
erase: O(logN)
find: O(logN)

map

map
map并不是哈希的方式实现的，而是和set一样以pair组成的红黑树，因此复杂度与set相同：
insert: O(logN)
erase: O(logN)
find: O(logN)

unordered_map

unordered_map
unordered_map才是以哈希实现的，具体的实现方式在下篇文章中具体说~，这篇只考虑复杂度：
insert: O(1)
erase: O(1)
find: O(1)

decltype

我们先说说decltype，这个语句的作用是获取某个变量的类型，但是当其对函数进行作用时，就表示获取一个函数的指针。
