C++中的排序

发表于 2018-11-20 | 分类于 C++ |
c++中使用了什么样的排排序算法提高效率的呢？

前几天对排序算法做了一个总结，引发了我的好奇，C++中到底使用了那种排序算法呢？

C++中的主要排序算法

我们首先分析一下各个算法的复杂度：

  1. 选择排序：时间：O(N^2)，空间：O(1)
  2. 插入排序：时间：最好O(N),平均最坏O(N^2)，空间：O(1)
  3. 希尔排序：时间：O(N^2)，空间：O(1)
  4. 归并排序：时间：O(NlogN)，空间：O(N)
  5. 快速排序：时间：最好平均O(NlogN)，空间：O(1)
  6. 堆排序： 时间：O(NlogN)，空间：O(1)

既然为了追求效率，那么就要从时间复杂度为O(NlogN)的算法中做选择了。归并排序时间复杂度很稳定，可是在大数据比较时会消耗太多的空间；快速排序复杂度并不稳定；堆排序虽然看上去很好，但是要知道进行堆排序首先需要生成堆的步骤！
这样一看似乎派苏排序的缺点好能说的过去，而且可以用一些方法对快排的最坏情况进行优化。
因此，C++就是使用快排作为排序函数的基础！并且为了防止最坏情况的发生，C++中采用了three medium方式进行优化，每次从左中右三个元素中的中间元素作为哨兵进行排序

快排的问题

只采用快排就可以了么？并不是！快排存在着什么样的问题呢？

  1. 快排算法中还存在一个问题：快排是递归算法啊！我们都知道，递归算法的递归调用虽然在时间复杂度上没有体现，但是消耗的时间比非递归的要长很多，可是快排没法携程非递归的形式，这就使得在数据量很少的情况下快排算法的时间性能并不优秀（长度为3的数组要递归3次！还不如直接比较来得快呢！）
  2. 虽然使用了three medium来进行改进，但是还是不能排除最坏情况的发生啊！

递归带来的损失

C++对排序算法进行了一步优化操作：定义一个thredhold，当待排序的元素个数小于这个阈值时，就以插入排序进行排序，否则就以快排的方式进行排序。那么为什么在这里选择插入排序呢？原因很简单，就是因为插入排序在数组基本有序的情况下排序排的很快啊！
并且，C++中的插入排序使用了一个trick：首先与第一个元素进行比较，再按照插入排序的方式依次向前比较。这样做的好处是：依次向前比较时无需检查是否越界的问题，这虽然是一个小优化（只比原版的插入排序少了一个比较），但是再数据量很大的时候，这种比较次数带来的时间性能上的改进就能体现出来了。

如何缓解最会情况的发生

快速排序中的最坏情况无论如何也无法完全避免（就算直接random shuffle也可能就变成了最坏情况了，而且还需要占用空间，总有电子不好的时候）那么C++中的排序算法提出了一个下策：实在不行就用堆排序呗~
那用什么来进行这种“实在不行”的程度划分呢？答案就是预先根据待排元素的个数给出一个期望值(log2N)，如果递归次数超过了这个期望值，就把剩下的问题变成堆排序了~
